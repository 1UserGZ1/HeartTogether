HeartTogether（血条共享）
一款 Minecraft Forge 模组（1.20.1），让玩家之间绑定并共享生命值、伤害吸收护盾、治疗效果，甚至同生共死——完美适配合作挑战、团队 PvP 或角色扮演场景。

📖 概述
HeartTogether 是一个轻量级的服务端模组，允许两名玩家 完全共享同一个生命池。任何伤害、吸收（例如附魔金苹果或药水效果）或治疗都会在绑定玩家之间即时同步。若其中一方死亡，另一方也会随之而去——并配有专属的殉情消息提示。

本模组专为 Forge 1.20.1（所有 47.x.x 子版本）设计，客户端与服务端均完美支持，且无需额外依赖。

✨ 功能特性
生命共享 – 绑定玩家共享相同的当前生命值和最大生命值。伤害和治疗同时作用于双方。

伤害吸收同步 – 任何来源的伤害吸收（如附魔金苹果、吸收药水等）都能即时镜像给伙伴。

即时受击反馈 – 一方受到真实伤害时，双方都会播放受伤音效和原版红色闪烁/抖动动画。

死亡同步 – 若一方死亡，所有绑定玩家同时死亡。直接受伤者显示原版死亡信息，其他同伴额外显示殉情提示（例如“Alex 殉情了”）。

动态绑定/解绑 – 通过简单的指令随时建立或解除绑定，解绑时保留当前共享血量。

多人支持 – 每对绑定独立运作，一个玩家同时只能参与一个绑定组。

📝 指令列表
指令	说明
/heart bind <玩家1> <玩家2>	将两名玩家绑定，自此他们的生命、吸收和治疗保持同步。
/heart unbind <玩家1> <玩家2>	解除绑定，双方各自保留当前血量。
/heart list	查看所有当前绑定对；若无绑定则显示“无玩家绑定”。
注意： 所有指令需要 管理员权限（等级 2+）。

🛠 兼容性
Minecraft 版本： 1.20.1

Forge 版本： 47.x.x（任意子版本）

Java： 17（OpenJDK）

依赖项： 无（纯 Forge 模组）

📦 安装方法
从 Releases 下载最新 hearttogether-<version>.jar。

将 jar 文件放入 Forge 1.20.1 服务端或客户端的 mods/ 文件夹。

重启服务端或启动客户端。

以管理员身份使用 /heart 指令。
